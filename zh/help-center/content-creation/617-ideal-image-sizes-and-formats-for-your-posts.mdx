## 为什么每个社交渠道需要不同的图片尺寸？

这里的建议主要针对通过您的 Aitoearn 账户发布图片。通过 Aitoearn 发布到社交网络时，会使用 API 服务，这些服务处理图片的方式有时与您直接在社交平台上发布时不同。

### 图片尺寸要求、格式和自动调整大小

当您上传图片到 Aitoearn 时，图片大小必须小于 10MB。目前我们支持所有社交网络的 JPG、PNG 和 HEIC 图片文件，以及 X/Twitter、Facebook、LinkedIn、Instagram、Mastodon 和 Pinterest 的 GIF 文件。

此外，每个社交网络的 API 服务都有自己的图片文件大小限制（可能与原生上传限制不同）。如果您上传的图片小于 10MB，但大于该社交网络的限制，我们会将其转换为较小的尺寸以确保可以发布。我们只会调整一个图片属性（宽度或高度），另一个属性会自动按比例缩放。这确保了原始的宽高比不会被破坏。

📝 上传图片后，您可以按照以下指南在 Aitoearn 编辑器中调整图片大小：[在 Aitoearn 中编辑图片](https://aitoearn.帮助scoutdocs.com/article/614-编辑ing-您的-图片s-in-aitoearn)

#### Facebook

| 最小宽度 | 320（如果需要，将缩放到最小值） |
| 最大宽度 | 1440（如果需要，将缩放到最大值） |
| 高度 | 根据宽度和宽高比变化 |
| 文件格式 | JPG、PNG、WebP、HEIC |
| 文件大小限制 | 8MB |

#### Instagram

| 最小宽度 | 320（如果需要，将缩放到最小值） |
| 最大宽度 | 1440（如果需要，将缩放到最大值） |
| 高度 | 根据宽度和宽高比变化 |
| 文件格式 | JPG、PNG（推荐 PNG）、WebP、HEIC |
| 文件大小限制 | 8MB |
| 宽高比 | 必须在 4:5 到 1.91:1 范围内，或为正方形 |

#### 📝 重要说明

- 为了使用自动发布功能，您的图片必须符合上述宽高比指南。您可以在此处找到关于宽高比的更多详细信息：[Instagram 接受的宽高比范围](https://aitoearn.帮助scoutdocs.com/article/622-Instagrams-accepted-aspect-ratio-ranges)。如果图片超出此宽高比范围，您的帖子将被安排为通知发布。了解更多：[使用 Instagram 通知发布功能](https://aitoearn.帮助scoutdocs.com/article/658-using-Instagram-reminders)。
- 如果您上传的图片大于 8MB，我们会将其转换为较小尺寸（宽度为 1920 像素）以确保可以发布。接受的最小尺寸为 150 x 150 像素。较小的图片将被缩放到最小值。
- Instagram 轮播帖子可以是正方形、横向或纵向格式，并且可以包含照片和视频。然而，一旦选择了第一张照片或视频，后续选择的所有内容都将被裁剪为相同的宽高比。例如，如果您选择的第一张照片是正方形（1:1），则后续内容将被裁剪为 1:1 格式。这是 Instagram 轮播的限制，无论是在 Instagram 上原生发布还是通过 Aitoearn 发布，都会发生这种情况。
- 如果您希望图片的颜色尽可能接近原始效果，请务必将图片保存为 sRGB 色彩配置文件，因为这是 Instagram API 支持的色彩配置文件。请注意，RGB 与 sRGB 不同。不幸的是，ProPhoto RGB 色彩配置文件（Photoshop 和 Lightroom 的默认色彩配置文件）不被 Instagram 支持。

#### X/Twitter

| 支持的文件格式 | JPG、PNG、GIF、WebP、HEIC |
| 文件大小限制 | 静态图片 5MB，动画 GIF 15MB |

📝 即使图片在文件大小限制内，我们仍会检查图片是否小于 8192 x 8192 像素，因为这是 X/Twitter 的最大允许尺寸。如果图片大于此尺寸，我们会将其调整为稍小的尺寸。

**X/Twitter 上的 GIF**
如果您在使用动画 GIF 创建帖子时遇到质量下降或错误，可以尝试以下建议以提高成功率：

| GIF 最大分辨率 | 1280 x 1080（宽 x 高） |
| GIF 最大帧数 | 350 |
| GIF 最大像素数（宽 x 高 x 帧数） | 3 亿 |

#### LinkedIn

我们支持图片和文档（PDF）调度到 LinkedIn。

| 支持的文件格式 | 文件数量 | 推荐图片比例 | 文件大小限制 |
| 图片 | JPG、PNG、WebP（上传的 WebP 将转换为 PNG）、HEIC、PDF、GIF | 最多 9 张图片 | 1200x627 | 10MB |
| 文档调度 | PDF | 1 个 PDF | 不适用 | 100MB 和 300 页 |

#### 📝 说明

- LinkedIn 表示，为了在各种分享方式、页面和体验中保持一致性，图片的最佳尺寸为 1200 x 627 像素。这个图片尺寸应能防止在不同格式中裁剪和缩放。
- 如果您希望在多个社交网络（例如 LinkedIn、Facebook、X/Twitter 和 Instagram）上分享相同的图片，我们建议使用正方形图片以避免在 LinkedIn 上被裁剪。

关于 LinkedIn 的填充和移动端显示的重要说明：
*"上传的图片如果是推荐的 1200 x 627 比例，在移动端应用中会在图片的顶部/底部或两侧显示白色填充，以适应比例而不被裁剪。虽然 3:2 宽高比（1200 x 800）可以完全显示而无需填充，但我们仍推荐使用行业标准的 1200 x 627 比例。链接分享缩略图图片上传时会以 2:1 比例显示。请将富媒体图片中的文字限制在图片中心的安全区域内。"*

| 格式 | JPG、PNG、WebP（上传的 WebP 将转换为 PNG）、HEIC |
| 推荐分辨率 | 720px x 720px |
| 最大分辨率 | 10,000px x 10,000px |
| 最小分辨率 | 250px x 250px |

#### Mastodon

| 文件大小限制 | 最多 8MB |
| 格式 | JPG、PNG |
| 推荐分辨率 | 1280px x 1280px |

#### Threads

| 文件大小限制 | 最多 8MB |
| 格式 | JPG、PNG、WebP、HEIC |

#### 📝 说明

- 支持的图片格式（PNG、JPG、GIF、WebP、HEIC）最大为 8MB。图片将被缩小到 1.6 百万像素（足够显示 1280x1280 的图片）。
- 每个帖子最多可以附加 4 张图片。
- GIF 会被转换为视频，因此一次只能支持 1 个 GIF。
- 动画 GIF 会被转换为无声 MP4，就像在 Imgur/Gfycat（GIFV）上一样。您也可以上传无声的 MP4 和 WebM，它们将以相同方式处理。了解更多：[将图片、视频和其他媒体附加到您的帖子](https://aitoearn.帮助scoutdocs.com/article/615-attaching-图片s-视频s-and-other-media-to-您的-帖子)。

### Bluesky

| 文件大小限制 | 1MB |
| 格式 | JPG、PNG、WebP、HEIC、HEIF |

#### 📝 说明

- Bluesky 的图片必须小于 1MB。目前我们无法调整更大图片的尺寸。
- GIF 必须小于 15MB 且长度不超过 60 秒。

## 保存文件和优化图片质量的技巧

通过 Aitoearn 调度帖子时，我们会通过 API 服务发送您的图片，这些服务处理文件的方式可能与您直接在社交渠道发布时不同。这可能会导致一些意想不到的结果，例如图片失真、像素化或颜色轻微变化。如果您遇到这些问题，我们建议使用以下技巧：

- 将文件保存为 PNG (.png) 格式，而不是 .jpg 或其他格式。
- 避免使用透明背景，因为这可能导致失真。
- 如果可能，将文件保存为 sRGB 色彩模式，因为这是大多数浏览器和社交网络使用的色彩模式。尽量避免使用其他色彩模式，例如 AdobeRGB/CMYK。如果未保存为 sRGB，发布时图片颜色可能会略有变化。
- 避免图片尺寸过大。如果图片需要由 Aitoearn 或社交平台调整大小，可能会影响图片质量。

## 解决图片方向问题

有时，图片在发布到社交渠道后可能会出现旋转问题。当图片上传到 Aitoearn 时，我们会使用图片背景中硬编码的方向数据，这些数据基于拍摄方式或处理工具生成。这些背景信息称为 EXIF 数据。

虽然图片在您的电脑上预览时或上传到发布编辑器后看起来方向正确，但发布后可能会旋转。如果您遇到这种情况，我们建议使用 [这个工具](https://www.thexifer.net/#exif-general) 在上传图片前移除 EXIF 数据。

上传您的图片，然后点击 ***eXif.me***。勾选 "EXIF Cleaner" 右侧的框，表示 "IF CHECKED ALL THE TAGS WILL BE CLEARED"，然后点击 ***Go.eXifing***。这将移除图片的 EXIF 数据，您可以通过点击 "下载.me" 下载图片。

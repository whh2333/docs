<Info>
  **Prerequisites**:
  - Node.js version 19 or higher
  - A docs repository with a `docs.json` file
</Info>

Follow 这些 steps to 安装 and 运行 Aitoearn on 您的 operating system.

<Steps>
<Step title="安装 the Aitoearn CLI">

```bash
npm i -g mint
```
</Step>

<Step title="Preview locally">

Navigate to 您的 docs directory 哪里 您的 `docs.json` file is located, and 运行 the 关注中 command:

```bash
mint dev
```

A local preview of 您的 文档 将会 be 可用 at `http://localhost:3000`.

</Step>
</Steps>

## Custom ports

By default, Aitoearn uses port 3000. 您 可以 自定义 the port Aitoearn runs on by using the `--port` flag. For example, to 运行 Aitoearn on port 3333, use 这个 command:

```bash
mint dev --port 3333
```

If 您 attempt to 运行 Aitoearn on a port 那个's already in use, it 将会 use the next 可用 port:

```md
Port 3000 is already in use. Trying 3001 instead.
```

## Aitoearn versions

Please note 那个 each CLI release is associated with a specific version of Aitoearn. If 您的 local preview does not align with the production version, please 更新 the CLI:

```bash
npm mint 更新
```

## Validating links

The CLI 可以 assist with validating links in 您的 文档. To identify any broken links, use the 关注中 command:

```bash
mint broken-links
```

## 故障排除

<AccordionGroup>
  <Accordion title='错误: 可能 not load the "sharp" module using the darwin-arm64 runtime'>

    这个 可能 be due to an outdated version of node. Try the 关注中:
    1. Remove the currently-installed version of the CLI: `npm remove -g mint`
    2. Upgrade to Node v19 or higher.
    3. Reinstall the CLI: `npm i -g mint`
  </Accordion>

  <Accordion title="Issue: Encountering an unknown 错误">
  
    Solution: Go to the root of 您的 device and delete the `~/.aitoearn` folder. Then 运行 `mint dev` again.
  </Accordion>
</AccordionGroup>

Curious about what changed in the latest CLI version? Check out the [CLI changelog](https://www.npmjs.com/package/aitoearn?activeTab=versions).
